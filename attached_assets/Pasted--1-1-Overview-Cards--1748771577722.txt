## 1. 대시보드 상단(헤더) 영역

1. **전체 요약 카드(Overview Cards)**
    - **총 게시(총 발행) 건수**: 오늘/이번 주/이번 달 기준으로 자동 또는 수동으로 발행된 게시물 수
    - **승인 대기 중 콘텐츠**: 현재 승인 대기 상태인 항목의 총 개수 (클릭 시 상세 목록으로 이동)
    - **오류 발생 건수**: 지난 24시간/이번 주 동안 발생한 API 호출 실패, AI 생성 실패 등 오류 이벤트 수
    - **전체 A/B 테스트 그룹 수**: 현재 진행 중인 A/B 테스트 그룹 수 및 완료된 그룹 수
    - **오늘의 트렌드 키워드**: 최신 시장 트렌드 분석에서 도출된 핵심 키워드(예: `#여행`, `#뷰티`, `#음식`)를 3~5개 정도 카드 형태로 노출
2. **실행 상태 알림(Notifications)**
    - **자동 워크플로우 실행 상태**: “오늘 09:00 자동 워크플로우 정상 실행됨” 또는 오류 알림 (“AI 텍스트 생성 실패 등”)
    - **중요 이벤트 팝업∙토스트**:
        - “새로운 콘텐츠 3건 생성 완료 – 승인 필요”
        - “A/B 테스트 그룹 ‘그룹_20250601’ 승자 결정 완료”
        - “TikTok API Rate Limit 경고: 잠시 후 재시도 필요”

## 2. 콘텐츠 관리(Content Management) 섹션

1. **승인 대기 목록(Pending Approval Table)**
    - **테이블 컬럼**
        1. 콘텐츠 ID
        2. 생성 일시
        3. 베트남어 제목(제목 텍스트 미리보기)
        4. 미디어 타입(이미지/영상) 및 미리보기 썸네일
        5. 자동화 모드(Auto vs Manual)
        6. 트렌드 키워드 요약(간략 텍스트)
        7. 상태(Approval Status)
        8. “승인” / “거절” 버튼
    - **기능**
        - **검색/필터**:
            - ‘자동화 모드(Auto/Manual)’, ‘생성 날짜’, ‘미디어 타입(이미지/영상)’, ‘트렌드 반영 여부’ 등의 필터 적용 가능
        - **미리보기 팝업**:
            - 행 클릭 시 팝업 또는 사이드 슬라이드아웃으로 “전체 베트남어/한국어 텍스트”, “AI 생성 원본 프롬프트”, “미디어(이미지/영상) 클립”을 상세 확인
        - **대량 승인/거절**:
            - 체크박스 선택 후 “일괄 승인” 또는 “일괄 거절” 버튼 제공
2. **콘텐츠 상태 현황(Content Status Summary)**
    - **도넛 차트(Doughnut Chart)**:
        - 승인 상태별 비율(Approved vs Pending vs Rejected vs Scheduled)
    - **진행 단계별 카운트**:
        - “텍스트 생성 완료 → 미디어 생성 완료 → 승인 대기 → 게시 대기 → 게시 완료 → 성과 수집 완료” 각 단계에 속한 콘텐츠 수를 카드 형태로 표시
    - **예약 게시 대기(Scheduled Posts)**
        - 곧 발행 예정인 콘텐츠 리스트(스케줄 시간, 플랫폼, 자동/수동 모드)
        - 필요 시 “취소” 또는 “시간 변경” 버튼 제공
3. **콘텐츠 상세 검색/관리(Control Panel)**
    - **검색창**: 키워드(제목/해시태그) 입력 시 즉시 필터링
    - **고급 필터**:
        - 생성 소스(Source Type: ‘AI 기본 생성’, ‘뉴스 요약 기반’, ‘댓글 분석 기반’, ‘트렌드 반영’ 등)
        - A/B 테스트 여부(‘A/B 테스트 대상’, ‘일반 콘텐츠’)
        - 플랫폼별 타겟(‘Facebook’, ‘TikTok’, ‘Instagram’ 등)
    - **컨텐츠 수정**:
        - 텍스트/해시태그 직접 수정 가능(수정 후 “AI 재생성”을 통해 새로운 버전 생성)
        - “미디어 다시 생성” 버튼(클릭 시 AI 이미지/영상 재호출)

## 3. 게시(Posting) 모니터링 섹션

1. **플랫폼별 게시 현황(Platform Posting Status)**
    - **카드 형태**로 각 플랫폼(Facebook, TikTok, Instagram, Threads, Twitter)별
        - **오늘 게시 건수**
        - **성공 건수 / 실패 건수**
        - **실패율(%)**
        - **마지막 오류 메시지 요약(있을 경우)**
    - **카드 클릭 시 팝업**:
        - 최근 5건의 게시 내역(ID, 콘텐츠 ID, 상태, 게시 시간, 에러 메시지(실패 시) 등 상세 표
2. **실시간 로그 실시간 업데이트(Live Logs)**
    - **스크롤 가능한 콘솔 뷰**로 실시간으로 전송되는 “AI 호출 로그”, “플랫폼 API 호출 로그”, “오류 로그” 등을 보여줌
    - **필터링**:
        - “로그 레벨(INFO, WARN, ERROR)”, “플랫폼(Facebook, TikTok 등)”, “워커 ID”별로 필터링
3. **예약 게시 관리(Scheduled Posts)**
    - **테이블**:
        - 콘텐츠 ID, 스케줄 일시, 대상 플랫폼, 자동화 모드, 승인 상태, 예약 상태(Active/Paused)
        - “예약 편집” 버튼(일시 변경), “예약 취소” 버튼 제공

## 4. 성과(Insights) 분석 섹션

1. **기본 성과 KPI 카드**
    - **총 도달(Reach) / 조회수(Views)**:
        - 지난 7일, 지난 30일 기준으로 플랫폼별 총합
    - **총 반응(Engagement)**: 좋아요, 댓글, 공유, 저장 등 합산(플랫폼별 주요 지표 기준)
    - **평균 참여율(주당/월당 평균)**: “Engagement ÷ Reach” 백분율
2. **플랫폼별 세부 성과 표(Platform Performance Table)**
    - **컬럼**:
        1. 콘텐츠 ID
        2. 게시 일시
        3. 플랫폼(Facebook, Instagram, TikTok, Twitter)
        4. Reach (도달)
        5. Engagement (좋아요+댓글+공유 등)
        6. Views (영상 조회수)
        7. 클릭(CTR, 클릭 수 또는 클릭률)
        8. ROI 지표(광고비/매출 연동 시)
    - **필터/검색**:
        - 기간(오늘, 7일, 30일, 맞춤 날짜 구간)
        - 플랫폼별, A/B 테스트 여부별, 캠페인별 필터 가능
    - **열 숨기기/표시 제어**: 사용자가 원하는 지표만 보여주도록 토글
3. **시계열 차트(Trends Over Time)**
    - **Line Chart 선택 옵션**(탭 전환 또는 드롭다운)
        - “일별 도달 추이”
        - “주별 조회수 추이”
        - “월별 Engagement 추이”
    - **드래그 가능한 날짜 범위 선택**(Zoom In/Out)
    - **툴팁**: 마우스 오버 시 각 날짜 포인트의 세부 지표 나타남
4. **콘텐츠 유형별 비교(Pie/Bar Chart)**
    - **파이 차트**:
        - “이미지 vs 영상” 기여도(총 조회수, 총 Engagement)
    - **막대 그래프**:
        - “플랫폼별 평균 Engagement” 비교
        - “A/B 테스트 승자 vs 비승자 평균 성과” 비교
5. **A/B 테스트 전용 성과 대시보드**
    - **테스트 그룹 리스트**: 간단한 표(그룹 ID, 생성 일시, 변형 수, 승자 상태)
    - **개별 그룹 클릭 시 상세 화면**:
        - **테이블**: 변형별 “플랫폼별 Reach/Views/Engagement” 수치
        - **비교 차트**:
            - “바 차트”로 변형 A vs B vs C의 조회수 비교
            - “라인 차트”로 시간 경과에 따른 성과 추이(예: 게시 후 24시간 간 조회수 증가 곡선)
        - **승자 표시**: 우승 변형 옆에 “🏆” 아이콘 및 “승자!” 태그 표시
        - **다음 액션**: “승자 게시물 청중 재타겟팅” 또는 “비승자 삭제” 버튼

## 5. A/B 테스트 관리 섹션

1. **A/B 테스트 개요 카드**
    - **진행 중인 테스트 수**: 현재 대기 중∙진행 중∙완료 상태별 수치
    - **최근 완료된 테스트**: 지난 7일간 완료된 그룹 ID(클릭 시 상세 분석 화면으로 링크)
2. **테스트 설정 위저드(Test Setup Wizard)**
    - **1단계: 원본 콘텐츠 선택**
        - 제목 검색 기능으로 후보 콘텐츠 검색 → 선택
    - **2단계: 변형 수 및 유형 지정**
        - “텍스트 변형 개수(예: 2~3)”, “미디어 변형 개수(예: 2)”, “추가 변형 옵션”(예: “제목+이미지 복합 변형”)
    - **3단계: 게시 스케줄 및 플랫폼 선택**
        - “변형 A는 오늘 10:00, 변형 B는 오늘 10:05 게시”처럼 개별 예약
        - 대상 플랫폼 체크박스(복수 선택 가능)
    - **4단계: 성과 비교 기준 설정**
        - “TikTok 조회수 1위 → 승자”, “Facebook Engagement(좋아요+댓글+공유) 1위 → 승자” 등
    - **5단계: 확인 및 시작**
        - 요약 정보(원본 콘텐츠, 변형 개수, 스케줄, 비교 기준 등) 표출 → “시작” 버튼
3. **실시간 테스트 진행 모니터링(Real-time Monitoring)**
    - **게시 대기 중 변형 목록**:
        - 변형 이름, 예정 게시 시각, 대상 플랫폼, 현재 상태(대기/게시 중/게시 완료)
    - **게시 완료 변형 성과**:
        - 게시 직후 1시간, 6시간, 24시간 기준 성과 지표 실시간 업데이트(자동 새로고침)
    - **예상 우승 추정치**:
        - “현재까지 데이터 기반 예측 승자” 표시(예: TikTok Views가 많이 나오고 있으면 “현재 변형 A 우세” 태그)
    - **경고 알림**:
        - “변형 B 게시 실패 – 재시도 필요”, “API Rate Limit 도달 예상” 등

## 6. 시장 트렌드 분석(Trend Insights) 섹션

1. **실시간 트렌드 키워드 클라우드(Word Cloud)**
    - 지난 24시간 수집된 키워드를 크기 비례로 표시
    - 주요 10개 키워드 강조(클릭 시 관련 뉴스/댓글 원문 보기)
2. **트렌드 요약 카드(Trend Summary Cards)**
    - “최근 인기 주제 3가지”: 간단한 문장으로 “#여행 콘텐츠가 급상승 중 → 여행 관련 게시물 우선 기획 권장” 등
    - “급감 키워드 2가지”: “#패션 키워드 관심 감소 → 우선순위 하향 조정” 등
3. **월별/주별 트렌드 변화 차트(Line Chart)**
    - 특정 키워드(예: #뷰티, #음식 등) 검색량(또는 언급량) 추이
    - 비교하여 “키워드 A vs 키워드 B” 차트 표시
4. **트렌드 기반 콘텐츠 추천(Recommendation)**
    - AI가 추천하는 “이 트렌드에 맞춘 다음 주제 3가지 제안” 리스트
    - 추천된 주제를 클릭하면 “해당 주제로 바로 콘텐츠 생성” 버튼 활성화 (해당 콘텐츠 생성 API 호출)

## 7. 사용자 상호작용 자동 응답(Engagement Bot) 섹션

1. **최근 댓글/메시지 수신 현황**
    - **테이블**:
        - 플랫폼(Facebook, Instagram), 콘텐츠 ID, 댓글 ID, 유저명, 댓글(메시지) 요약, 수신 시간, 자동 응답 여부
    - **필터 기능**:
        - “미응답 댓글만 보기”, “부정적 키워드 포함 댓글만 보기” 등
2. **자동 응답 로그(Reply Log)**
    - **테이블**:
        - 댓글 ID, 유저명, 수신 댓글 원문, AI가 생성한 응답 문장, 응답 성공/실패 상태, 응답 시간
    - **데이터 다운로드(CSV)**: AI 응답 이력 전체 다운로드 버튼
3. **응답 실패 항목(Failed Replies)**
    - “응답 실패 사유(예: 토큰 만료, API 오류)”를 함께 보여주는 전용 목록
    - “재시도” 버튼을 클릭하면 AI 재호출 및 게시 재시도
4. **실시간 Webhook 상태(Health Check)**
    - “Facebook Webhook 연결 정상/오류”, “Instagram Webhook 연결 정상/오류”
    - 마지막 Webhook 이벤트 수신 시간 표시
    - “Webhook 재연결” 버튼(필요 시)

## 8. 설정(Settings) 화면

1. **스케줄링 설정**
    - **콘텐츠 생성 스케줄**: “매일 09:00” 등 Cron 표현식 입력 또는 드롭다운 선택
    - **성과 수집 스케줄**: “매일 10:00” 등
    - **A/B 테스트 지연 설정**: 예) 변형 간 최소 2분 지연 등
2. **A/B 테스트 기준 설정**
    - **플랫폼별 우선순위 가중치**:
        - Facebook Engagement 가중치(예: 0.3)
        - TikTok Views 가중치(예: 0.5)
        - Instagram Reach 가중치(예: 0.2)
    - **최종 승자 기준**: “가중치 합산 점수 기준” 또는 “특정 플랫폼 우선 순위” 선택
3. **알림 채널 설정**
    - **Slack Webhook URL** 입력
    - **관리자 이메일 주소**(복수 입력)
    - **알림 대상 이벤트**:
        - “콘텐츠 생성 완료”
        - “승인 요청 도착”
        - “게시 성공/실패”
        - “A/B 테스트 종료”
        - “AI 오류 발생” 등 각각 토글 선택
4. **플랫폼 API 키 관리**
    - **Facebook/Instagram**: Page ID, Access Token 입력 테스트
    - **TikTok**: Client Key, Client Secret, Redirect URI, Test API 연결 상태 표시
    - **Twitter**: API Key, API Secret, Bearer Token 입력 테스트
    - **Cloudinary**: Cloud Name, API Key, API Secret 테스트
5. **AI 모델 파라미터**
    - **Google AI Ultra 설정**:
        - 온도(Temperature), 최대 토큰(Max Output Tokens), Top-p 등 슬라이더 또는 입력 필드
    - **Stability AI 설정**: CFG 스케일, 이미지 해상도, 샘플 수 조정
    - **RunwayML 설정**: 영상 길이(초), 해상도(720×1280, 1080×1920 등)
6. **데이터 저장소 선택(최종)**
    - **Google Sheets 사용 여부 토글**
    - **데이터베이스 연결 문자열**(RDBMS or MongoDB) 입력
    - **백업 주기 설정**: “DB 스냅샷 매일 / Google Sheets CSV 내보내기 주간” 등
7. **보안 및 인증**
    - **Webhook 검증 토큰**: Facebook/Instagram Webhook 검증용 토큰 설정
    - **관리자 접근 제어**: IP 화이트리스트, 2단계 인증(2FA) 활성화 여부
    - **로그 레벨 설정**: DEBUG, INFO, WARN, ERROR 선택

## 9. 에러 모니터링 및 시스템 헬스 체크

1. **실시간 시스템 상태 배너(System Health Banner)**
    - **API 응답 시간 평균**(AI 호출, 플랫폼 API 각 평균 응답 시간)
    - **최근 1시간 오류율**(백분율)
    - **서버 CPU/메모리 사용량**(자체 서버 호스팅 시)
2. **오류 로그 대시보드(Error Log Dashboard)**
    - **테이블**: timestamp, 모듈 이름(예: AI, SNS, Scheduler), 오류 메시지 요약, 심각도 레벨
    - **필터**:
        - 기간(예: “지난 24시간”, “지난 7일”)
        - 모듈별 필터(“AI 텍스트 생성 오류”, “Facebook API 오류” 등)
        - “미해결 오류만 보기” 토글
3. **재시도 큐 상태(Retry Queue Status)**
    - **큐에 쌓여있는 작업 수**(예: “재시도 대기 중인 포스팅 3건”)
    - **최근 재시도 이력**: 작업 ID, 콘텐츠 ID, 시도 횟수, 마지막 시도 시간
    - **“전체 재시도” 버튼**: 수동으로 큐에 있는 모든 작업 재시도 트리거
4. **Webhook 상태(Health Check)**
    - **Facebook/Instagram Webhook**: 최종 이벤트 수신 시간, 정상 연결 여부
    - **AI API 상태**: Google AI Ultra, Stability AI, RunwayML 등 외부 서비스 연결 상태(정상/지연/에러)
    - **DB/Google Sheets 연결 상태**: 연결 테스트 버튼(“DB 연결 테스트” 클릭 시 “성공/실패” 표시)
