## **2. 콘텐츠 관리(Content Management) 섹션**

이 섹션은 AI 및 자동화 시스템을 통해 생성되거나 관리되는 콘텐츠의 라이프사이클을 관리합니다. 생성된 콘텐츠를 검토하고, 상태를 추적하며, 필요에 따라 수정하고 검색하는 기능을 제공합니다.

### **1. 승인 대기 목록(Pending Approval Table)**

- **기능 정의 및 목적**: 사용자가 시스템에 의해 생성된 콘텐츠를 검토하고 게시 여부를 결정하는 핵심 워크플로우 단계입니다. 승인이 필요한 모든 콘텐츠를 효율적으로 관리하고, 대량의 콘텐츠를 빠르게 처리할 수 있도록 지원합니다.
- **세부 사양 및 동작 방식**:
    - 데이터는 '승인 대기(Pending)' 상태인 콘텐츠만 표시합니다.
    - 테이블 컬럼에 정의된 정보를 각 행에 표시합니다. 베트남어 제목은 길이가 길 경우 일정 글자 수 이후에는 생략 부호(...)로 표시하고 호버 시 전체 텍스트를 툴팁으로 보여줍니다.
    - 미디어 미리보기 썸네일은 이미지 또는 영상의 첫 프레임을 작게 표시합니다. 영상의 경우 재생 버튼 아이콘을 오버레이할 수 있습니다.
    - 각 행 끝에는 '승인', '거절' 버튼이 제공됩니다. 버튼 클릭 시 해당 콘텐츠의 상태가 즉시 변경되며 (Pending -> Approved 또는 Pending -> Rejected) 테이블에서 사라집니다.
    - 테이블 상단 또는 측면에 검색/필터 컨트롤이 제공됩니다. 필터 조건 적용 시 테이블 데이터가 즉시 업데이트됩니다.
    - 테이블 행 클릭 시, 해당 콘텐츠의 상세 정보를 보여주는 **미리보기 팝업 또는 사이드 슬라이드아웃**이 나타납니다. 이 미리보기에는 전체 베트남어 및 한국어 텍스트 (번역 결과 포함), AI 생성에 사용된 원본 프롬프트, 전체 미디어 클립(영상 재생 기능 포함)이 포함됩니다.
    - 테이블 각 행의 좌측에 체크박스가 제공되며, 체크박스 선택 후 테이블 상단의 '일괄 승인', '일괄 거절' 버튼을 클릭하면 선택된 모든 콘텐츠의 상태를 일괄 변경합니다. 일괄 작업 시 진행 상태를 표시하는 로딩 인디케이터가 제공됩니다.
- **사용자 인터페이스(UI) 요소**:
    - **데이터 테이블**: 승인 대기 중인 콘텐츠 목록을 표시하는 주요 컴포넌트입니다. 컬럼 헤더 클릭 시 정렬 기능이 제공될 수 있습니다.
    - **체크박스**: 각 행 좌측 및 테이블 헤더에 전체 선택/해제용 체크박스.
    - **검색 입력란**: 제목 또는 키워드로 검색.
    - **필터 드롭다운/체크박스 그룹**: 자동화 모드, 생성 날짜 범위, 미디어 타입, 트렌드 반영 여부 등을 선택하는 컨트롤.
    - **버튼**: 각 행에 '승인', '거절'. 테이블 상단에 '일괄 승인', '일괄 거절'.
    - **미리보기 팝업/사이드 슬라이드아웃**: 콘텐츠 상세 정보를 표시하는 오버레이 UI. 내부에는 텍스트 영역 (베트남어/한국어 전환 탭 또는 동시 표시), 이미지/영상 플레이어, AI 프롬프트 텍스트 영역 등으로 구성됩니다.
- **사용자 상호작용(UX)**:
    - 사용자는 테이블을 스캔하며 승인 대상 콘텐츠를 확인합니다.
    - **행 클릭**하여 상세 미리보기를 통해 콘텐츠 내용을 깊이 검토합니다.
    - 각 행의 버튼을 클릭하거나 **체크박스 선택 후 일괄 버튼 클릭**으로 승인/거절 액션을 수행합니다.
    - 검색 입력 또는 필터 옵션 선택/입력으로 목록을 **필터링**하여 원하는 콘텐츠만 확인합니다.
    - 컬럼 헤더 클릭으로 목록을 **정렬**합니다.
- **데이터 요구사항**:
    - 콘텐츠 데이터베이스에서 상태가 'Pending'인 모든 콘텐츠 데이터: 콘텐츠 ID, 생성 일시, 베트남어/한국어 제목 및 전체 텍스트, 미디어 파일 경로/URL, 미디어 타입, 자동화 모드, 연관 트렌드 키워드, AI 생성 원본 프롬프트, 현재 상태(Pending).
    - 미디어 파일 서버 또는 CDN으로부터의 접근 권한.
- **기술적 고려사항/제약조건**:
    - 테이블 **데이터 로딩 성능**: 승인 대기 콘텐츠가 많을 경우를 대비하여 페이지네이션(Pagination) 및 서버 사이드 필터링/정렬이 필수입니다.
    - **미디어 미리보기 성능**: 특히 영상 클립 로딩 및 재생이 끊김 없이 부드럽게 이루어져야 합니다.
    - **일괄 작업 처리**: 대량의 데이터를 처리할 때 서버 부하를 고려하여 비동기 처리 또는 배치(Batch) 처리를 적용할 수 있습니다.
    - **베트남어 지원**: 베트남어 텍스트의 정확한 표시 및 처리가 중요합니다. 한국어/베트남어 전환 또는 병기 기능은 번역 서비스 연동을 고려할 수 있습니다.
- **예외 처리 및 오류 시나리오**:
    - 승인/거절 요청 중 서버 오류 발생 시 사용자에게 실패 메시지를 표시하고, 상태 변경이 반영되지 않았음을 알립니다.
    - 미리보기 팝업에서 미디어 파일 로딩 실패 시 대체 이미지나 오류 메시지를 표시합니다.
    - 일괄 승인/거절 중 일부 항목에서 오류가 발생한 경우, 성공한 항목과 실패한 항목을 구분하여 사용자에게 결과를 요약하여 알립니다.

### **2. 콘텐츠 상태 현황(Content Status Summary)**

- **기능 정의 및 목적**: 전체 콘텐츠의 현재 상태 분포 및 워크플로우 진행 단계를 시각적으로 보여줌으로써, 콘텐츠 처리 파이프라인의 병목 구간이나 특정 상태에 집중된 콘텐츠 양을 빠르게 파악할 수 있도록 합니다.
- **세부 사양 및 동작 방식**:
    - **도넛 차트**: 시스템 내 모든 콘텐츠의 현재 상태(Approved, Pending, Rejected, Scheduled, Posted 등)별 개수를 집계하고 전체에서 차지하는 비율을 도넛 차트로 시각화합니다. 각 섹션에 마우스 오버 시 해당 상태의 정확한 개수와 비율을 툴팁으로 표시합니다.
    - **진행 단계별 카운트**: 콘텐츠가 생성부터 게시, 성과 수집까지 거치는 주요 워크플로우 단계별로 현재 해당 단계에 속해 있는 콘텐츠 수를 집계하여 카드 형태로 표시합니다. 각 단계는 미리 정의된 상태 그룹을 의미합니다 (예: '텍스트 생성 완료' 단계는 '텍스트 생성 완료' 상태이지만 아직 '미디어 생성'은 안 된 콘텐츠).
    - **예약 게시 대기 목록**: 상태가 'Scheduled'인 콘텐츠 중 특정 기간(예: 향후 24시간 또는 7일) 내에 게시 예정인 콘텐츠 목록을 간략히 표시합니다. 여기서는 목록 확인 및 기본적인 액션(취소/시간 변경) 링크 또는 버튼을 제공합니다. (상세 관리는 '게시 모니터링' 섹션의 예약 게시 관리와 연계 또는 통합될 수 있습니다).
- **사용자 인터페이스(UI) 요소**:
    - **도넛 차트**: 색상 구분된 도넛 형태로 상태 비율을 표시합니다. 차트 중앙에는 전체 콘텐츠 수를 표시할 수 있습니다. 차트 범례(Legend)를 함께 제공하여 각 색상이 어떤 상태를 나타내는지 명확히 합니다.
    - **상태 카드**: 각 워크플로우 단계를 나타내는 작은 카드 그룹. 각 카드에는 단계 이름과 해당 콘텐츠 수가 표시됩니다 (예: **[텍스트 생성 완료] 150**).
    - **예약 목록 컴포넌트**: 곧 게시될 콘텐츠 목록을 보여주는 간략한 테이블 또는 리스트 형태. 각 항목에는 콘텐츠 ID, 스케줄 시간, 대상 플랫폼, 모드 정보가 포함됩니다. 각 항목 옆에 **"취소"**, **"시간 변경"** 등의 액션 링크 또는 버튼이 포함될 수 있습니다.
- **사용자 상호작용(UX)**:
    - 사용자는 차트와 카드를 통해 시스템 내 콘텐츠 흐름을 시각적으로 파악합니다.
    - 차트 섹션 또는 상태 카드를 **클릭**하여 해당 상태 또는 단계에 속한 콘텐츠의 상세 목록(예: '콘텐츠 상세 검색/관리' 테이블에서 해당 상태/단계로 필터링된 결과)으로 이동할 수 있습니다.
    - 예약 목록에서 항목을 **클릭**하여 상세 정보를 확인하거나, 버튼을 클릭하여 **예약 취소** 또는 **시간 변경**을 수행합니다 (시간 변경 시 별도 팝업/모달 제공).
- **데이터 요구사항**:
    - 콘텐츠 데이터베이스에서 모든 콘텐츠의 현재 상태 및 워크플로우 진행 단계 정보.
    - 예약 게시 모듈로부터 예약된 콘텐츠 목록 및 스케줄 정보.
    - 데이터 집계를 위해 콘텐츠 데이터베이스에 대한 효율적인 쿼리가 필요합니다.
- **기술적 고려사항/제약조건**:
    - 데이터 집계는 실시간에 가깝게 업데이트되어야 합니다. 대규모 데이터셋의 경우 집계 쿼리의 **성능 최적화**가 중요하며, 캐싱이나 집계 테이블 활용을 고려합니다.
    - 차트 라이브러리 선택 시 데이터 업데이트 및 애니메이션 성능을 고려합니다.
    - '예약 게시 대기' 목록은 '게시 모니터링' 섹션의 '예약 게시 관리' 테이블과 데이터 소스를 공유해야 하며, 기능 중복을 최소화하거나 서로 연계되도록 설계해야 합니다.
- **예외 처리 및 오류 시나리오**:
    - 데이터 집계 중 오류 발생 시 차트와 카드가 '데이터 로드 실패' 또는 'N/A'로 표시됩니다.
    - 예약 목록 로딩 실패 시 해당 영역을 비워두거나 오류 메시지를 표시합니다.
    - 예약 취소/시간 변경 실패 시 사용자에게 오류 메시지를 표시합니다.

### **3. 콘텐츠 상세 검색/관리(Control Panel)**

- **기능 정의 및 목적**: 시스템 내 모든 콘텐츠를 포괄적으로 검색, 필터링하고, 필요에 따라 콘텐츠 내용(텍스트, 해시태그)을 수정하거나 AI를 통해 콘텐츠 일부를 재생성하는 기능을 제공합니다.
- **세부 사양 및 동작 방식**:
    - 사용자는 검색창에 키워드(제목, 해시태그, 내용 일부)를 입력하여 실시간 또는 입력 완료 후 검색 버튼 클릭으로 콘텐츠 목록을 필터링합니다.
    - **고급 필터** 영역에서는 생성 소스, A/B 테스트 여부, 플랫폼 타겟 등 다양한 조건으로 검색 결과를 정교하게 필터링할 수 있습니다. 여러 필터 조건을 조합하여 적용 가능합니다.
    - 검색/필터 결과는 테이블 또는 리스트 형태로 표시되며, 각 항목에는 콘텐츠 ID, 상태, 제목 등 주요 정보가 간략히 표시됩니다.
    - 결과 목록에서 특정 콘텐츠를 선택(클릭)하면, 해당 콘텐츠의 상세 정보를 확인하고 **수정**할 수 있는 에디터 화면 또는 팝업/모달로 이동합니다.
    - 에디터 화면에서는 베트남어 및 한국어 텍스트, 해시태그 등을 직접 수정할 수 있습니다. 수정 후 '저장' 버튼을 클릭하면 변경 사항이 반영됩니다.
    - 에디터 화면 내에서 'AI 텍스트 재생성', '미디어 다시 생성' 버튼을 제공합니다. 'AI 텍스트 재생성' 클릭 시 현재 텍스트 및 프롬프트 기반으로 AI 호출하여 새로운 텍스트 초안을 받아옵니다. '미디어 다시 생성' 클릭 시 기존 프롬프트 및 메타 정보 기반으로 AI 이미지/영상 생성 모듈을 다시 호출합니다. 이 기능들은 새로운 버전 생성 또는 현재 콘텐츠 수정 방식에 따라 다르게 구현될 수 있으며, 일반적으로는 AI 재생성 시 새로운 초안을 제시하고 사용자가 선택/수정하는 워크플로우를 따릅니다.
- **사용자 인터페이스(UI) 요소**:
    - **검색 입력란**: 키워드 입력 필드.
    - **고급 필터 섹션**: 확장/축소 가능한 영역 내에 드롭다운, 체크박스, 라디오 버튼 등 다양한 필터 컨트롤 포함.
    - **결과 목록**: 검색 및 필터 조건에 맞는 콘텐츠를 표시하는 테이블 또는 카드 목록.
    - **콘텐츠 에디터 화면/모달**: 선택된 콘텐츠의 상세 내용을 표시하고 수정할 수 있는 인터페이스. 텍스트 입력 영역 (베트남어/한국어 탭 또는 분리), 미디어 미리보기 영역, 해시태그 입력/수정 필드, **'저장'**, **'취소'**, **'AI 텍스트 재생성'**, **'미디어 다시 생성'** 버튼 포함.
- **사용자 상호작용(UX)**:
    - 사용자는 검색 키워드 입력 및 필터 옵션 선택으로 원하는 콘텐츠를 빠르게 찾습니다.
    - 결과 목록에서 항목을 **클릭**하여 콘텐츠 상세 정보 확인 및 수정 모드로 진입합니다.
    - 에디터에서 텍스트, 해시태그를 **직접 수정**합니다.
    - 필요 시 **AI 재생성 버튼을 클릭**하여 AI 기능을 활용해 콘텐츠를 업데이트합니다.
    - 수정 내용을 **저장**하거나 변경 사항을 **취소**합니다.
- **데이터 요구사항**:
    - 콘텐츠 데이터베이스에 저장된 모든 콘텐츠 메타데이터 및 본문 데이터 (텍스트, 해시태그, 미디어 경로, 생성 소스, A/B 테스트 정보, 플랫폼 타겟, 상태 등).
    - AI 서비스(텍스트 생성, 이미지/영상 생성)와의 연동을 위한 API 호출 기능.
- **기술적 고려사항/제약조건**:
    - **검색 성능**: 전체 콘텐츠를 대상으로 하는 검색 및 필터링이 빠르고 효율적으로 동작해야 합니다. Full-text search 인덱싱, 데이터베이스 쿼리 최적화 또는 검색 엔진(예: Elasticsearch) 도입을 고려할 수 있습니다.
    - **콘텐츠 수정 워크플로우**: 수동 수정 또는 AI 재생성 시 콘텐츠 **버전 관리** 정책을 정의해야 합니다 (예: 수정 시 새 버전 생성, 승인된 콘텐츠 수정 제한 등).
    - **AI 서비스 연동**: AI 모델의 응답 시간 및 API 호출 제약(Rate Limit)을 고려하여 비동기 처리 및 사용자에게 진행 상태 표시가 필요합니다. 특히 베트남어 텍스트 생성/처리를 위한 AI 모델 지원이 중요합니다.
- **예외 처리 및 오류 시나리오**:
    - 검색 결과가 없는 경우 사용자에게 '검색 결과 없음' 메시지를 표시합니다.
    - 콘텐츠 저장 실패 시 사용자에게 오류 메시지를 표시하고 재시도 옵션을 제공합니다.
    - AI 재생성 호출 실패 시 이유(예: AI 서버 오류, 입력값 문제)와 함께 실패 메시지를 표시합니다.
    - 사용자가 수정 권한이 없는 콘텐츠를 수정하려 할 경우, 권한 부족 메시지를 표시하고 저장 버튼을 비활성화합니다.
