# Role:
당신은 최고의 그룹웨어 기획 전문가이자 UI/UX 설계자입니다. 사용자의 업무 효율을 극대화할 수 있는 직관적이고 기능적인 인터페이스를 설계하는 데 특화되어 있습니다.

# Background:
현재 'LN Partners 그룹웨어'의 '마케팅 관리' 메뉴에 포함될 **'마케팅 계획 상세 페이지'**를 개발하고 있습니다. 제공된 페이지 이미지와 구성 요소(HTML, 텍스트)를 보면, 계획의 개요와 목표(OKRs)를 조회하는 기능은 이미 구현되어 있습니다.

[현재 페이지 핵심 구성 요소]

계획 개요: 계획명, 기간, 담당자, 전체 진행률, 설명, 타겟 고객, 핵심 메시지, 주요 채널
목표 설정 (OKRs):
Objective (목표): 달성률 표시 (예: Z세대 인지도 확보 63%)
Key Results (핵심 결과):
수치 기반 KR (예: 틱톡 팔로워 5만 달성)
비율 기반 KR (예: 브랜드 인지도 20% 증가)
체크리스트 기반 KR (예: UGC 콘텐츠 수집 하위 태스크)
코멘트: 담당자 간 커뮤니케이션
이제 이 페이지에 1) 계획 내용을 수정하는 기능과 2) 진행 상황을 요약하여 보고하는 기능을 추가하려고 합니다.

# Task:
주어진 '마케팅 계획 상세 페이지'의 구성 요소를 바탕으로, 아래 두 가지 핵심 기능에 대한 상세 기획안을 UI/UX 설계 관점에서 작성해 주세요.

요청사항 1: '마케팅 계획 수정' 기능 기획
사용자가 '수정' 버튼을 눌렀을 때, 현재 페이지에서 바로 내용을 편집할 수 있는 '인라인 수정(Inline Edit)' 방식으로 기획해 주세요. 별도의 수정 페이지로 이동하지 않습니다.

UI/UX 설계:

'수정' 버튼을 클릭하면, 페이지 상단에 '저장'과 '취소' 버튼이 나타납니다.
수정 가능한 모든 텍스트 필드와 데이터 영역이 편집 가능한 상태(input box, text area, dropdown 등)로 변경됩니다.
항목별 수정 방식 상세 제안:
계획명, 설명, 핵심 메시지: 텍스트 입력 필드로 변경
기간: Date Picker UI로 변경
담당자: 사내 직원 검색 및 선택이 가능한 드롭다운 또는 자동 완성 검색창으로 변경
주요 채널: 태그(Tag) 입력 방식으로, 채널 추가/삭제 가능하도록 설계
OKRs:
Objective, Key Result 텍스트: 인라인 텍스트 편집
수치 기반 KR (틱톡 팔로워): 현재 값(32,000)만 수정할 수 있도록 하고, 목표 값(50,000)은 고정. 현재 값 수정 시 진행률 자동 계산.
체크리스트 기반 KR (UGC 콘텐츠 수집): 체크박스 on/off 및 하위 태스크 추가/삭제/편집 기능
프로세스:

사용자가 '저장' 버튼을 누르면, 변경 사항이 시스템에 즉시 반영되고 다시 조회 모드로 전환됩니다.
'취소' 버튼을 누르면, 모든 변경 사항이 원복되고 조회 모드로 돌아갑니다.
요청사항 2: '마케팅 계획 보고서' 생성 기능 기획
사용자가 '보고서' 버튼을 클릭하면, 현재까지의 성과를 시각적으로 명확하게 보여주는 보고서 팝업 또는 페이지가 생성되어야 합니다. 보고서는 차트와 시각화 자료를 중심으로 구성해 주세요.

보고서 구성 요소:

보고서 제목: (예: "2025년 1분기 마케팅 계획 진행 보고서")
기본 정보: 계획명, 보고 기간, 담당자, 보고서 생성일
전체 진행 현황 (Overall Progress):
계획 전체 진행률(65%)을 보여주는 도넛 차트(Doughnut Chart) 또는 게이지 차트.
핵심 목표(Objectives)별 달성 현황:
각 Objective의 달성률을 비교할 수 있는 막대 차트(Bar Chart).
(예: 'Z세대 인지도 확보' 63%, '온라인 매출 증대' 66%를 막대그래프로 시각화)
핵심 결과(Key Results) 상세:
수치 기반 KR (틱톡 팔로워): 목표치와 현재 달성치를 한눈에 보여주는 게이지 차트(Gauge Chart). (예: 50,000명 중 32,000명 달성)
비율 기반 KR (브랜드 인지도): 목표치(20%) 대비 달성률(75%)을 나타내는 중첩 원형 차트 또는 단계별 진행률 바(Progress Bar).
체크리스트 기반 KR (UGC 콘텐츠 수집): 완료된 태스크와 미완료 태스크 수를 보여주는 간단한 표 또는 파이 차트.
주요 코멘트 요약: 보고서 생성 시점까지의 주요 코멘트를 2~3개 발췌하여 포함.
기능:

보고서 상단에 '인쇄' 및 'PDF로 내보내기' 버튼을 배치하여 사용자가 오프라인 문서로 활용할 수 있도록 합니다.
# Output Format:

위 요청사항들을 바탕으로, 각 기능에 대한 UI/UX 설계안을 **마크다운(Markdown)**을 사용하여 작성해 주세요.
특히 '보고서' 기능은, 마크다운 테이블이나 ASCII 아트를 활용하여 차트가 포함된 보고서 레이아웃 예시를 시각적으로 표현해 주세요.
각 기능의 플로우와 사용자 인터랙션에 대한 설명을 명확하게 기술해 주세요.
